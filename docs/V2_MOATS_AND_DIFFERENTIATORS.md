# V2 Moats And Differentiators

| # | Item | One-line justification | Other coding agents clearly have this? |
|---|---|---|---|
| 1 | Session Record / Export | Turn each AI work session into a portable, inspectable asset instead of disposable chat history. | Partial |
| 2 | Session Replay / Resume Anywhere | Resume the same work state under a different model or environment without rebuilding context. | No clear evidence |
| 3 | “Why This Model?” Explanation Layer | Make every routing and fallback decision legible so the control plane is trustworthy. | Partial |
| 4 | Branchable Sessions as First-Class Work Units | Let AI work behave more like version-controlled thinking than linear chat. | Partial |
| 5 | Execution Checkpoints | Save and restore workflow state so experiments and recovery do not require starting over. | Yes, partially |
| 6 | Policy Profiles / Routing Profiles | Give teams named operating modes for cost, speed, privacy, and approval posture. | Partial |
| 7 | Team Memory Packs / Shareable Context Capsules | Make project knowledge reusable across sessions, teammates, and model workers. | Partial |
| 8 | Workproof Reports | Produce a manager-readable report of what happened, what changed, and what it cost. | Partial |
| 9 | Cross-Model Continuity Benchmark | Measure how well work survives a model handoff instead of only measuring single-model quality. | No clear evidence |
| 10 | Agent Flight Recorder | Record every model hop, command, artifact, and policy decision for replay and audit. | Partial |
| 11 | Session Handoff Packages | Hand an active AI workstream from one engineer to another without losing state. | No clear evidence |
| 12 | “Replace the Worker” Button | Swap the active model mid-task while keeping the same workflow state and control logic. | No clear evidence |
| 13 | Time-Travel Diff of Thought and Output | Show exactly what changed in reasoning and output before and after a model switch. | No clear evidence |

## Notes

- `Partial` means adjacent features exist, but not the full Xyntra-style control-plane version.
- `No clear evidence` means no strong official evidence was found in current competitor docs reviewed during analysis.
